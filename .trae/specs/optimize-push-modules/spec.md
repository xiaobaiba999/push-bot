# 推送模块系统性优化 Spec

## Why
当前推送项目存在功能冗余（每日情话与彩虹屁重叠）、纪念网站在微信环境被屏蔽无法访问、缺乏交互式AI对话能力、天气模块缺少紫外线指数等实用信息。需要系统性优化以提升用户体验和功能完整性。

## What Changes
- **BREAKING** 移除"每日情话"模块（loveSentence），因与"彩虹屁"模块内容完全重叠
- 修复纪念网站在微信环境中的访问问题，改用可绕过微信屏蔽的链接策略
- 新增AI对话模式，接入免费AI模型实现企业微信群内交互问答
- 天气模块增加紫外线指数显示及防护建议

## Impact
- Affected code: `index.js`, `input.js`, `functions/loveSentence.js`, `functions/weather.js`, `functions/robotPush.js`, `package.json`
- 新增文件: `functions/aiChat.js`, `functions/callbackHandler.js`
- 删除文件: `functions/loveSentence.js`（两处）
- 需要新增依赖: `@anthropic-ai/sdk` 或其他免费AI API客户端
- 需要阿里云FC新增HTTP触发器配置（用于接收企业微信回调）
- 需要阿里云OSS或TableStore配置（用于对话历史存储）

**Reason**: 与"彩虹屁"模块功能完全重叠，两者均调用同一API（api.shadiao.pro/chp）获取情话类文案，造成推送内容冗余
**Migration**: 彩虹屁模块已全面涵盖情话内容，无需额外迁移。移除后推送内容更简洁

#### 需删除的文件和引用
- `functions/loveSentence.js`（根目录和push/push/两处）
- `input.js` 中 `loveSentence` 配置项及导出
- `index.js` 中 `loveSentence` 相关的解构、条件判断、内容拼接逻辑

## MODIFIED Requirements

### Requirement: 纪念网站链接展示
当前实现直接输出 `https://www.liyaoyao.top`，在微信内置浏览器中触发"已停止访问该网页"的安全拦截。

#### Scenario: 微信环境访问纪念网站
- **WHEN** 用户在微信中点击推送中的纪念网站链接
- **THEN** 应能正常打开网站，不出现"已停止访问"提示

#### 修改方案
微信屏蔽域名的根因通常是：域名被举报、SSL证书异常、内容违规、未在微信开放平台备案等。从推送代码层面，采用以下策略：

1. **链接格式优化**：将直接URL改为引导性文本+可复制URL格式，避免微信自动识别为外链跳转
2. **备用访问方式**：提供二维码或搜索引导作为备选
3. **Markdown卡片消息**：改用企业微信markdown消息类型，链接以更友好的方式展示

> **注意**：域名被微信屏蔽的根本解决方案需要在网站侧操作（申请微信域名解封、完善SSL、内容合规审查等），推送代码只能做展示层面的优化。

### Requirement: 天气模块 - 增加紫外线指数

#### Scenario: 正常天气推送包含紫外线信息
- **WHEN** 定时推送执行，天气模块开启
- **THEN** 推送内容中包含紫外线强度等级（1-5级）和对应防护建议

#### 紫外线等级与防护建议映射
| 等级 | 指数范围 | 描述 | 防护建议 |
|------|---------|------|---------|
| 1 | 0-2 | 最弱 | 无需特别防护 |
| 2 | 3-5 | 弱 | 适当涂抹防晒霜 |
| 3 | 6-7 | 中等 | 外出需涂防晒霜、戴帽子 |
| 4 | 8-10 | 强 | 避免长时间户外活动，必须防晒 |
| 5 | 11+ | 极强 | 尽量避免外出，必须全面防护 |

#### 数据来源
和风天气API `/v7/indices/1d` 接口，type=5（紫外线指数），该接口在免费订阅中可用。

#### Scenario: 紫外线API请求失败
- **WHEN** 紫外线指数接口请求超时或返回错误
- **THEN** 天气推送仍正常展示气温、风况等信息，紫外线部分显示"暂无数据"，不影响整体推送

## ADDED Requirements

### Requirement: AI对话模式
系统应提供企业微信群内的AI对话交互功能，用户可通过@机器人发起提问，机器人调用AI模型生成回答并以推送消息方式回复。

#### 架构设计
```
用户@机器人 → 企业微信回调 → 阿里云FC(HTTP触发器) → AI模型API → 企业微信Webhook推送回复
```

#### Scenario: 用户在群内@机器人提问
- **WHEN** 用户在企业微信群内@机器人发送消息（如"@小助手 今天适合出门吗"）
- **THEN** 机器人在3秒内通过推送消息回复AI生成的回答

#### Scenario: 多轮对话上下文理解
- **WHEN** 用户连续发送多条相关消息
- **THEN** AI能理解上下文语境，给出连贯的回答（保留最近5轮对话历史）

#### Scenario: 对话历史查看
- **WHEN** 用户发送"历史记录"或"查看对话"
- **THEN** 机器人返回最近10条对话摘要

#### 技术实现要点

1. **回调处理函数** `functions/callbackHandler.js`
   - 验证企业微信回调签名（URL验证+消息验证）
   - 解析用户消息内容
   - 调用AI对话模块
   - 通过Webhook推送回复

2. **AI对话模块** `functions/aiChat.js`
   - 接入免费AI模型（推荐DeepSeek API，免费额度充足，中文能力强，响应快）
   - 实现对话上下文管理（最近5轮）
   - 对话历史存储（阿里云OSS，按用户ID分文件）
   - 响应超时控制（3秒超时，超时返回默认提示）

3. **阿里云FC配置**
   - 新增HTTP触发器，路径如 `/callback`
   - 触发器需配置为公网可访问
   - 函数内存建议256MB，超时15秒

4. **企业微信机器人配置**
   - 在群机器人设置中配置回调URL指向FC的HTTP触发器地址
   - 设置Token和EncodingAESKey用于消息验证

5. **免费AI模型选择**
   - **DeepSeek API**：免费额度500万tokens/月，响应速度<2秒，中文能力优秀
   - 备选：智谱AI（GLM-4-flash免费额度）、月之暗面（Kimi免费额度）
   - API Key配置在 `input.js` 中

#### 限制与约束
- 企业微信群机器人回调功能需要机器人在群内被@才能触发
- 对话历史存储依赖阿里云OSS，需额外开通和配置
- 3秒响应目标取决于AI API实际延迟，网络波动时可能超时
- 免费AI API有调用频率限制，高并发时可能触发限流
